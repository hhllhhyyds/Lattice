# Task 10.1: Support Multiple Parallel Tool Calls (Sequential Execution)

## 关联 Issue

- Closes #27: OpenAI client silently drops parallel tool calls

## 问题描述

当前实现中，OpenAI 和 Anthropic 客户端都只处理第一个工具调用，静默丢弃其他并行工具调用：

- **OpenAI**：`parse_response()` 中使用 `.next()` 只提取第一个
- **Anthropic**：`response_to_decision()` 中只返回第一个 ToolUse

这导致：
- Agent 无法完成 LLM 期望的所有操作
- 用户得到不完整的结果
- 没有错误或警告，难以调试

## 目标

实现多工具调用的**串行执行**支持，让所有工具调用都能被执行并记录结果。

**非目标**：
- 并行执行（留待未来优化）
- 修改 LLM 提示词让其不返回多个工具调用

## 设计方案

### 1. 类型系统变更

#### 1.1 新增 `Decision::MultiToolCall` 变体

**文件**：`crates/core/src/llm.rs`

```rust
pub enum Decision {
    Thinking { reasoning: String },
    ToolCall { tool: String, params: serde_json::Value },
    
    /// Multiple tool calls to be executed sequentially.
    /// All tools will be executed regardless of individual failures.
    MultiToolCall { calls: Vec<ToolCallRequest> },
    
    FinalAnswer { answer: String },
}

/// A single tool call request within a multi-tool-call decision.
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct ToolCallRequest {
    /// Unique identifier for this tool call (from LLM provider).
    pub id: String,
    /// Name of the tool to invoke.
    pub tool: String,
    /// Tool parameters as JSON.
    pub params: serde_json::Value,
}
```

### 2. LLM Protocol 层变更

#### 2.1 更新 `response_to_decision()`

**文件**：`crates/llm-protocol/src/parse.rs`

**当前逻辑**：
```rust
LLMResponse::Mixed { blocks } => {
    // 只返回第一个 ToolUse
    for block in &blocks {
        if let ContentBlock::ToolUse { name, input, .. } = block {
            return Ok(Decision::ToolCall { ... });  // ← 只返回第一个
        }
    }
}
```

**新逻辑**：
```rust
LLMResponse::Mixed { blocks } => {
    // 收集所有 ToolUse
    let tool_uses: Vec<_> = blocks.iter()
        .filter_map(|b| match b {
            ContentBlock::ToolUse { id, name, input } => Some((id, name, input)),
            _ => None,
        })
        .collect();
    
    if tool_uses.is_empty() {
        // 没有工具调用，返回文本
        let text = extract_text(&blocks);
        return Ok(Decision::FinalAnswer { answer: text });
    } else if tool_uses.len() == 1 {
        // 单个工具调用，保持向后兼容
        let (id, name, input) = tool_uses[0];
        return Ok(Decision::ToolCall {
            tool: name.clone(),
            params: input.clone(),
        });
    } else {
        // 多个工具调用
        let calls = tool_uses.into_iter()
            .map(|(id, name, input)| ToolCallRequest {
                id: id.clone(),
                tool: name.clone(),
                params: input.clone(),
            })
            .collect();
        return Ok(Decision::MultiToolCall { calls });
    }
}
```

### 3. OpenAI 客户端变更

#### 3.1 更新 `parse_response()`

**文件**：`crates/llm-openai/src/client.rs`

**当前逻辑**：
```rust
if let Some(tool_calls) = msg.tool_calls {
    if let Some(tc) = tool_calls.into_iter().next() {  // ← 只取第一个
        return Ok(LLMResponse::ToolUse { ... });
    }
}
```

**新逻辑**：
```rust
if let Some(tool_calls) = msg.tool_calls {
    if tool_calls.is_empty() {
        return Err(LLMError::InvalidResponse("empty tool_calls array".into()));
    }
    
    if tool_calls.len() == 1 {
        // 单个工具调用 - 返回 ToolUse（向后兼容）
        let tc = tool_calls.into_iter().next().unwrap();
        let input = parse_arguments(&tc.function.arguments);
        return Ok(LLMResponse::ToolUse {
            id: tc.id,
            name: tc.function.name,
            input,
        });
    } else {
        // 多个工具调用 - 返回 Mixed
        let blocks: Vec<ContentBlock> = tool_calls.into_iter()
            .map(|tc| {
                let input = parse_arguments(&tc.function.arguments);
                ContentBlock::ToolUse {
                    id: tc.id,
                    name: tc.function.name,
                    input,
                }
            })
            .collect();
        return Ok(LLMResponse::Mixed { blocks });
    }
}
```

### 4. ControlLoop 变更

#### 4.1 处理 `Decision::MultiToolCall`

**文件**：`crates/runtime/src/control_loop.rs`

```rust
Decision::MultiToolCall { calls } => {
    info!(count = calls.len(), "LLM requested multiple tool calls");
    let parent_id = events.last().map(|e| e.event_id);
    
    // 1. 记录所有 ToolCallRequested 事件
    let mut request_ids = Vec::new();
    for call in &calls {
        let req_event_id = self.store.append_event(
            session_id,
            EventPayload::ToolCallRequested {
                tool: call.tool.clone(),
                params: call.params.clone(),
            },
            Actor::LLM,
            parent_id,
        ).await?;
        
        events.push(Event {
            event_id: req_event_id,
            session_id,
            timestamp: chrono::Utc::now(),
            actor: Actor::LLM,
            payload: EventPayload::ToolCallRequested {
                tool: call.tool.clone(),
                params: call.params.clone(),
            },
            parent_event_id: parent_id,
        });
        
        request_ids.push(req_event_id);
    }
    
    // 2. 串行执行所有工具，分别记录结果或错误
    for (call, req_event_id) in calls.iter().zip(request_ids) {
        info!("executing tool: {}", call.tool);
        
        match self.tools.execute(&call.tool, call.params.clone()).await {
            Ok(result) => {
                info!("tool execution succeeded: exit_code={}", result.exit_code);
                let result_event_id = self.store.append_event(
                    session_id,
                    EventPayload::ToolCallResult {
                        stdout: result.stdout.clone(),
                        stderr: result.stderr.clone(),
                        exit_code: result.exit_code,
                    },
                    Actor::Sandbox,
                    Some(req_event_id),
                ).await?;
                
                events.push(Event {
                    event_id: result_event_id,
                    session_id,
                    timestamp: chrono::Utc::now(),
                    actor: Actor::Sandbox,
                    payload: EventPayload::ToolCallResult {
                        stdout: result.stdout,
                        stderr: result.stderr,
                        exit_code: result.exit_code,
                    },
                    parent_event_id: Some(req_event_id),
                });
            }
            Err(e) => {
                warn!("tool execution failed: {}", e);
                let error_str = e.to_string();
                let error_event_id = self.store.append_event(
                    session_id,
                    EventPayload::ToolCallError {
                        error: error_str.clone(),
                    },
                    Actor::Sandbox,
                    Some(req_event_id),
                ).await?;
                
                events.push(Event {
                    event_id: error_event_id,
                    session_id,
                    timestamp: chrono::Utc::now(),
                    actor: Actor::Sandbox,
                    payload: EventPayload::ToolCallError { error: error_str },
                    parent_event_id: Some(req_event_id),
                });
            }
        }
    }
    
    info!(count = calls.len(), "all tool calls completed");
}
```

## 错误处理策略

### 原则：全部执行，分别记录

- **一个工具失败不影响其他工具**：继续执行剩余工具
- **所有结果都记录**：成功记录 `ToolCallResult`，失败记录 `ToolCallError`
- **LLM 看到完整信息**：下次决策时能看到所有工具的结果
- **与单工具调用一致**：错误处理逻辑保持一致

### 事件序列示例

**场景**：3 个工具调用，第 2 个失败

```
Event 1: UserMessage "比较 file1.txt, file2.txt, file3.txt"
Event 2: LLM Decision (MultiToolCall with 3 calls)
Event 3: ToolCallRequested (call_1: cat file1.txt, parent=2)
Event 4: ToolCallRequested (call_2: cat file2.txt, parent=2)
Event 5: ToolCallRequested (call_3: cat file3.txt, parent=2)
Event 6: ToolCallResult (call_1: success, parent=3)
Event 7: ToolCallError (call_2: "file not found", parent=4)
Event 8: ToolCallResult (call_3: success, parent=5)
Event 9: LLM Decision (基于所有结果继续)
```

## 风险与缓解

### 风险 1：向后兼容性
**风险**：现有代码可能依赖单工具调用行为
**缓解**：保持单工具调用返回 `Decision::ToolCall`，只在多工具时返回 `MultiToolCall`

### 风险 2：事件顺序
**风险**：事件记录顺序可能影响 LLM 理解
**缓解**：遵循"先记录所有请求，再记录所有结果"的顺序

### 风险 3：性能
**风险**：串行执行可能较慢
**缓解**：这是第一版实现，未来可以优化为并行执行
