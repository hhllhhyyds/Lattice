# 任务 15.1：BashTool 平台适配 — 根据平台提供不同的工具描述

## 目标

将 BashTool 改造为平台感知的工具，根据操作系统向 LLM 提供正确的工具名称和描述，避免 LLM 生成错误平台的命令。

## 分支

`feat/platform-aware-tool`

## 依赖

- 任务 05.1（Windows 兼容性支持）✅ 必须先完成
- 任务 15（工具系统）✅

## 背景

### 当前问题

BashTool 的描述是硬编码的：

```rust
ToolDescription {
    name: "bash".to_string(),
    description: "Execute a bash command in a sandboxed environment. \
         Use this for running shell commands, scripts, and system operations."
        .to_string(),
    // ...
}
```

**问题**：
1. 工具名称是 "bash"，但在 Windows 上实际使用 cmd.exe
2. 描述说 "bash command"，LLM 可能生成 Unix 命令（`ls`, `grep`）
3. 在 Windows 上，这些命令会失败（应该用 `dir`, `findstr`）
4. LLM 不知道当前平台，无法生成正确的命令

### 用户需求

1. ✅ 动态察觉当前平台（Windows/Linux）
2. ✅ Sandbox 中的命令依据平台变化（任务 05.1 已解决）
3. ⚠️ **BashTool 的描述需要告知 LLM 当前需要什么平台的命令**

### 阶段 3：TDD Phase 2 — 实现平台感知的描述

**文件**：`crates/tools/src/bash.rs`

**实现策略**：
使用条件编译在编译时生成平台特定的描述。

**代码修改**：

```rust
#[async_trait]
impl ToolExecutor for BashTool {
    fn description(&self) -> ToolDescription {
        #[cfg(unix)]
        {
            ToolDescription {
                name: "sh".to_string(),
                description: "Execute a Unix shell command in a sandboxed environment. \
                     Use Unix-specific commands like 'ls' (list files), 'cat' (read file), \
                     'echo' (print text), 'grep' (search text), 'find' (search files)."
                    .to_string(),
                parameters_schema: serde_json::json!({
                    "type": "object",
                    "properties": {
                        "command": {
                            "type": "string",
                            "description": "The Unix shell command to execute"
                        }
                    },
                    "required": ["command"]
                }),
            }
        }

        #[cfg(windows)]
        {
            ToolDescription {
                name: "cmd".to_string(),
                description: "Execute a Windows cmd.exe command in a sandboxed environment. \
                     Use Windows-specific commands like 'dir' (list files), 'type' (read file), \
                     'echo' (print text), 'findstr' (search text), 'where' (find files)."
                    .to_string(),
                parameters_schema: serde_json::json!({
                    "type": "object",
                    "properties": {
                        "command": {
                            "type": "string",
                            "description": "The Windows cmd.exe command to execute"
                        }
                    },
                    "required": ["command"]
                }),
            }
        }
    }

    // execute() 方法保持不变
    async fn execute(&self, params: serde_json::Value) -> Result<ExecutionResult, ToolError> {
        let command = params
            .get("command")
            .and_then(|v| v.as_str())
            .ok_or_else(|| {
                ToolError::InvalidParams(
                    "missing or invalid 'command' field (expected string)".to_string(),
                )
            })?;
        self.sandbox
            .execute(command, params.clone())
            .await
            .map_err(|e| ToolError::ExecutionFailed(e.to_string()))
    }
}
```

**新增内容**：

```markdown
## Platform-Aware Tools

BashTool is platform-aware and provides different tool descriptions based on the target platform:

- **Unix/Linux/macOS**: Tool name is `"sh"`, description includes Unix commands (ls, cat, grep)
- **Windows**: Tool name is `"cmd"`, description includes Windows commands (dir, type, findstr)

This ensures the LLM generates platform-appropriate commands. The platform detection happens at compile time using `#[cfg(unix)]` and `#[cfg(windows)]`.

### Example

On Unix:
```json
{
  "name": "sh",
  "description": "Execute a Unix shell command... Use commands like 'ls', 'cat', 'grep'..."
}
```

On Windows:
```json
{
  "name": "cmd",
  "description": "Execute a Windows cmd.exe command... Use commands like 'dir', 'type', 'findstr'..."
}
```
```
